# Overview
I want to be able to manage my milk from my local milkman
# Goals
A mobile UI for customers: Customers should be able to manage milk deliveries on a schedule
A mobile UI for milk deliverers: This should show the list of locations that you need to deliver to, and the orders for each stop.

# Specifications

## Customer Experience
### Login
The starting screen should ask a customer to sign in by scanning a QR code, or typing in a unique code. That QR code or unique code can be generated by a milk supplier from their mobile app.

### Persistent Session
If the customer has already logged in, persistent session state should be stored client side to ensure they remain logged in. It should be checked against the identity provider when the application is opened, but otherwise be transparent. Once a customer has logged in once, they should stay logged in permanently.

## Landing Page
When customers open the application they should see details of their next order.
If a customer's next order has passed the deadline for order amendments then it should be shown without the ability to amend it.

If the deadline for orders has passed then the user should see their order presented in a way where they can see what is coming (and any unavailable items)

If the deadline has _not_ passed then there should be quick options to increase or decrease the quantity of their various items. There should also be a link at the bottom of the component that takes them to the full edit page, in they want to add new items.

There should also be an option to "skip" the order, which cancels everything. If the deadline has passed this option should be disabled.

# Landing Page - New customers

If a customer has just signed up them they should be taken directly to a page to set up their orders. Firstly they should be presented with a list of days they want to subscribe to. This will depend on the availability of the supplier to cover that specific area. The products will also be specific to a supplier.

Once the customer has picked their days, they should then be taken to the next page where they can choose to select products that are delivered on those days. To simplify things, the default view should just assume that the customer wants to order the same thing on each day (e.g. two pints of whole milk every day).

If the customer wants to order something different for a day, then they should be able to "split off" a day and set up a different order for just that day.

# Edit Order

When you are on the edit order page you should be presented with a vertical list of the items that you have ordered, with quantities beside them. There should be + and - icons to alter the quantity. There should be a maximum quantity that can be supplied, after which the + icon greys out, the - icon should grey out when quantity is 1. There should be a trash icon that deletes the item from the order.

Additionally at the top of the page should be a free text box. When a customer enters some text into this, it should bring up a short autocompleted list of products that match for that supplier. Clicking on one of the items adds it to the current order, with quantity set to 1.

The supplier's name should be presented at the top of the list, along with the date the order will be delivered. If a customer is amending an existing order, then there should be a toggle at the top that can switch between "Change my order for XXX date" and "Change all my orders on XXX day"

There should be an option to attach a note to the order, as free text at the bottom of the list.
